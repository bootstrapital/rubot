Rubot builds the control layer AI features need.

Rubot is a Rails-native framework for agentic internal tools. It gives you typed tools, structured agents, resumable workflows, approvals, traces, and operator surfaces without forcing product logic out of your app.

Operational AI instead of prompt demos.

Keep workflows inspectable, approval-aware, and replayable. Build with real application boundaries instead of hiding behavior behind opaque agent loops.

Core Surface

Four primitives, one runtime.

Rubot stays small on purpose. Tools execute application actions. Agents produce structured reasoning. Workflows orchestrate ordered steps with pauses and resume points. Operations group feature logic for app-facing entry points.

Tool

Typed app actions

Write explicit, schema-backed service objects for the actions that touch your systems. 

class LookupAccount < Rubot::Tool
  input_schema { string :account_id }
  output_schema { string :status }
end
Agent

Structured reasoning

Keep prompts, model choice, and output contracts in one place instead of scattering them through controllers. 

class ReviewAgent < Rubot::Agent
  model "gpt-4.1-mini"
  output_schema { string :summary }
end
Workflow

Durable orchestration

Chain tool and agent steps, pause for review, then resume with a full event trail and public output snapshot. 

approval_step :manager_review,
              role: "ops_manager"

step :finalize
Build Order

Design the workflow before the model flourish.

The right unit of work is usually a governed workflow, not a free floating agent. Rubot makes the runtime state visible enough to debug and controlled enough to ship inside production systems.

01

Define the boundaries

Put real app actions in tools and use schemas so bad inputs fail fast.

  • Explicit inputs and outputs
  • Idempotent actions where possible
  • Runnable in plain Ruby or Rails
02

Orchestrate the decisions

Compose tool steps, agent steps, and approvals into one ordered execution path.

  • Pause for human review when risk changes
  • Resume from durable checkpoints
  • Persist useful state, not just final strings
03

Operate with evidence

Inspect events, replay runs, and evaluate behavior against fixtures before drift turns into product regressions.

  • Traceable events and approvals
  • Replay support for debugging
  • Eval harnesses for regression checks
Product UI stays in your app. Execution logic lives in tools, agents, workflows, and operations.
Why It Fits Rails

Rubot does not replace your application architecture. It gives you a runtime layer for the parts that need state, approvals, model access, and operator visibility.
